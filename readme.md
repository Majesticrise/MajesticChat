# 去中心化 P2P 聊天室 (P2P Chat Room)

基于 EasyTier 组网的去中心化 P2P 聊天程序，支持群聊、私聊、文件传输、历史同步与自动节点发现。

---

## 特性

- **去中心化组网**：通过 EasyTier 自动建立 P2P 虚拟网络，无需中心服务器。
- **即时通信**：
  - 群聊（全网广播）
  - 私聊（`/msg <昵称> <消息>`）
- **文件传输**：
  - 流式传输（逐块发送，支持大文件）
  - MD5 校验确保完整性
  - 传输进度显示
  - 支持取消传输（`/cancel`）
- **历史同步**：新节点加入后自动从其他节点同步历史消息（基于消息 ID）。
- **节点发现**：定时刷新对等节点列表，自动连接新节点。
- **心跳保活**：定时 Ping/Pong 检测连接状态，超时自动断开并重连。
- **消息持久化**：聊天记录（含私聊）保存在本地 SQLite 数据库，支持 `/history` 查看最近 20 条。
- **跨平台支持**：Windows / Linux / macOS（需适配 EasyTier 可执行文件）。

## 快速开始

### 前置要求

- Python 3.8+
- 下载 EasyTier 核心文件（`easytier-core` 和 `easytier-cli`）放置在程序同目录：
  - [EasyTier Releases](https://github.com/zhanghanyun/easytier/releases)（选择对应平台）
- 确保网络环境允许 UDP 打洞（中继服务器默认使用公共中继 `38.147.105.178:11010`）



---

## 工作原理
- 组网：程序启动 EasyTier 进程，加入指定的虚拟网络，获取虚拟 IP（如 10.x.x.x）。

- 节点发现：通过 easytier-cli peer 获取当前网络中的所有对等节点 IP。

- 连接建立：主动与发现的节点建立 TCP 长连接（端口 8888），交换握手消息（昵称）。

- 消息同步：连接建立后，双方交换各自的最后消息 ID，同步缺失的历史消息（分页传输，每批 50 条）。

- 消息广播：用户输入的消息通过 JSON 封装后广播给所有已连接的节点；私聊消息仅发送给指定节点。

- 文件传输：发送端计算文件 MD5，分块（64KB）Base64 编码后通过 JSON 消息发送；接收端校验 MD5 并保存。

- 心跳与重连：每 30 秒发送 Ping，若超时 60 秒未收到 Pong 则断开连接并触发重连（指数退避）。

## 注意事项
本程序默认使用公共中继服务器 38.147.105.178:11010，若需使用私有中继，请修改代码中的 relay_ip 和 relay_port（或后续版本支持配置文件）。

文件传输无大小限制，但建议网络带宽允许范围内使用；接收端未做文件大小限制，需注意磁盘空间。

运行于 Windows 时，建议使用 Windows Terminal 或 PowerShell 以获得完整的 Emoji 显示效果。

若遇到防火墙问题，请确保允许 8888 端口的入站连接。
